(Git Cheatsheet)

Befehl Funktion
git init Erstellt ein lokales git Repository
git add . Gibt an was alles in den commit rein soll
git commit -m “message” Hinterlegt einen Message/Commit
git push Schickt das lokale Repository in das Github Repository
git log --oneline Zeigt alle die bis dato erstellten Commits in Kurzform an
git branch “name” Erstellt einen neuen Branch
git branch -a Zeigt alle erstellten Branches an
git branch -d “name” Löscht den ausgewählten Branch
git checkout Wechseln zwischen den Branches
git pull Lädt die aktuelle Version des Repository von GitHub runter
git clone Kopiert ein fremdes Projekt von Github
git merge Ist dafür da um zwei Branches zusammenzuführen

Der Unterschied zwischen git und Github

  • Git wird zur Umsetzung von Befehlen in Github verwendet
  • GitHub ist eine Plattform die Git benutzt

Abbreviations

  • A - Added (This is a new file that has been added to the repository)
  • M - Modified (An existing file has been changed)
  • D - Deleted (a file has been deleted)
  • U - Untracked (The file is new or has been changed but has not been added to the repository yet)
  • C - Conflict (There is a conflict in the file)
  • R - Renamed (The file has been renamed)
  • S - Submodule (In repository exists another subrepository)
